Classification version of CORE

Version 0.9.15   

Last updated in Ljubljana, Slovenia on September, 23th, 2005

 CORE is an experimental learning system, which learns decision trees, random forests or evaluates the attributes. Some of its abilities are described in the papers at http://lkm.fri.uni-lj.si/rmarko/papers/

 

Decision trees

While building a decision tree the system, in each node, evaluates the attributes (with ReliefF, GainRatio, MDL, or any other built in heuristics). It builds decision trees in  the top-down manner and post prunes them (using m-estimate or  MDL pruning).

 

Random forests

The basic algorithm of Leo Breiman is implemented plus some improvements:

  • it alows selection of different attribute evaluati on heuristics, even several of them in one forest
  • probability estimation as averages of tree probab lities
  • weighted voting where weights are average margin of trees in local subspaces
  • regularization

 

Attribute evaluation

The most interesting evaluation heuristics is the algorithm ReliefF, which is capable of detecting conditional dependencies between the attributes. For each attribute it evaluates the probability that it separates different from similar near instances. Together with its adaptation for regression problems called RReliefF it provides an unified view to attribute estimation. See the paper Theoretical and Empirical Analyses of ReliefF and RReliefF (MLJ, 2003) and other papers at http://lkm.fri.uni-lj.si/rmarko/papers/

 

Data format

The system uses its own input format as well as the format of C4.5, C5, See5 and Cubist. Native data format is described in DataFormat.htm file.

 

Source code

The system is written in C++ and should compile without problems as console application with Microsoft, Borland or GNU C++ compiler. The top level definitions should be set in general.h file. The executable for 32bit Windows-Intel platform is included.

Linux

Uncomment #define UNIX in general.h file and compile the sources with:
>g++ *.cpp -O2 -o dire

 

Execution

Start the system by:

  • dire    # starts in user interface mode
  • dire o=parameter_file   # starts in user interface mode with parameters from file
  • dire o=parameter_file a=action  # executes specified action with parameters from file
  • dire o=parameter_file a=action keyword=value ... # executes specified action with parameters from file, where specified parameters override the ones in the file

 The action can be any of the following:

  • treeOnce - builds decision tree on one specified data split
  • treeAll - builds decision tree models on all data splits
  • rfOnce  -  builds random forest on one data split
  • rfAll - builds random forest on all data splits
  • estOnce - evaluates attributes with estimators from parameter file on one selected data split
  • estAll  - evaluates attributes with estimators from parameter file on all data splits.
  • data  - outputs characteristics of the data set

Keywords are the same as the ones in the parameter file.

An example of how to prepare the data is included. It is an Australian credit data set from UCI repository, and contains the following files:

  • credita.par - the file with parameters,
  • credita.dat - file with the data,
  • credita.dsc - description of the attributes,
  •  credita.01s,... credita.09s - definition of splits to training and testing data prepared for 10-fold cross-validation,
  • credita.cm - cost matrix

For example to test random forests with parameters from credita.par, except for attribute estimator:

> dire o=credita.par a=rfAll selectionEstimator=7
