Data format for CORE learning system

 

The data can be prepared in native data format or in format of C4.5, C5, See5 or Cubist. These formats are described in documents at http://www.rulequest.com. If you are using them do not change suffix of a file (.names, .data, .cost), use the files as they are. This file describes the native data format.

 

 

CORE and DIRE native data format

 

Every problem should have a name assigned to it, for example, credita or autompg. An extension determines the type of data it contains:

  • .dsc: description of  attributes, for example credita.dsc
  • .dat: the actual data in tabular form, for example credita.dat
  • .cm: cost matrix for classification (used only in DIRE (not neccessary), for example credita.cm

 

The definition of splits to training and testing data can also be prepared on files which should have names with .Ns extension, where N is the consequtive number of split (begining with 0),

for example credita.00s, credita.01s, ... To use this possibility the option splitSelection=0 (use splits to train/test data from files). This option and splits on files can be used with C5 data format as well.

 

Commments

# and % are comment characters. If they appear at the beggining of the line, that particular line is ignored.

 

 

Description of the attributes

should be contained in the file with .dsc suffix. The first line contains the number of desacriptions (variables, attributes). Then follows the description of all variables starting with the predicted one. Description of the variable consist of the variable name on one line and the next line contains its description which depends on the type of the attribute. If it is

  • discrete: it contains the number of values and then each value on a separate line. The order of values should be identical to their value in the data file e.g.,the first value should be transcribed as 1 in the data file and so on.
  • numeric: it should contaion zero and then two value meaning the difference of two values to be considered equal and different, respectively. If these two values are set to 0.0 then defaults from parameter file are used

 

 

The data

should be contained in the file with .dat suffix. The first line contains the number of instances. Then follows each instance on a separate line. The % sign cause the line to be

ignored from the sign till the end of line. The values for one instances should be in the order described in .dsc file, prediction being the first followed by attributes. The discrete values should be noted with their consecutive number. The values could be separated by comma, spaces or tabs.

 

 

The cost matrix

should be contained in the file with .cm suffix. Cost matrix is used only with DIRE (classification problems). Even in DIRE it is not neccessary to use costs. If you are working with

equal costs of missclasification (also called accuracy, zero-one loss, or error rate) there is no need for cost matrix. The matrix should contain cost entries in the matrix form. The matrix must be of the size C x C, where C is the number of class values. C[i,j] presents the cost of predicting j-th class value if true class value is i-th. One line in the file corresponds one line in the matrix.
